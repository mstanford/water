Introduction
  Water is a simple scripting language designed to perform a variety of tasks. Water is implemented in C# for use with .NET/Mono. Water allows easy access to .NET libraries and supports .NET datatypes. In addition to piggy-backing on .NET, Water is better suited for tasks where a csproj and an edit/compile development cycle don't add value. Water is useful for experimenting with algorithms, unit testing, performance testing, text processing, code generation, or just having fun!

Water expressions are evaluated by typing a command followed by a list of arguments. 
>print "a"
a

In Water all statements begin with an operator (a.k.a. prefix-notation). 
+ 1 1
 
 

Constants and Variables
  In Water there are two types of stored data: variables and constants. Constants are declared with the constant operator and variables with the var operator.

To reassign a variable use the assignment operator =. To set a property of a variable use the set operator.
 
 

Datatypes
  Water has primitive datatypes and composite datatype. Composite datatypes are lists and dictionaries composed of one or more primitive or composite datatypes. Comments begin with a semi-colon.

Primitives: 
null - Has no value.

boolean - true/false

integer - 0, 1, ... , n

float - 0.001, 3.14, 555.55

string - "Hello World!", "C:\\Temp"

identifier - n, person, print

Composites: 
list - (a b c)

dictionary - { a: "a" b: 5 }

Comments: 
; Comment
 
 

Conditional Statements
  Water supports 2 types of conditional evaluation if statements and switch statements. 
if true
    print "TRUE"
else
    print "FALSE"
end_if



swith n
    case 1
        print "1"
    case 2
        print "2"
    default
        print "unknown"
end_switch
 
 

Iterative Statements
  Water supports 3 types of loops: foreach, while, and for. 
foreach item in list
    print item
end_foreach



while true
    print "still going..."
end_while



for i from 1 to 10
    print i
end_for
 
 

Functions
  Functions can contain any number of statements that are executed together when the function is called. Functions must be defiend before they are called. The function... 
function print-hello who
    print "Hello, ${who}."
end_function
will evaluate its contents when called... 
>print-hello "world"
"Hello, world."
